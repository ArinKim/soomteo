# 프로젝트

숨터(soomteo) 프로젝트는 청소년 및 노약자를 위한 전화 상담 모바일 어플리케이션 제작.
어플리케이션이 너무 무겁거나 사용하기 번거로우면 안됨.

## 프론트엔드 작업 기초 기술 스택

React Native + Javascript
Tailwind CSS
shadcn/ui 컴포넌트
lucide-react 아이콘
Framer Motion 애니메이션
Recharts 차트

## 기본 프로젝트 구조

- 스타일, css 등은 분리해서 /styles 안에 넣을 것
- 아래의 구조는 예시이므로 깔끔하게 분리 가능하다면 임의로 변경해도 괜찮음

```
/app                  # expo-router 라우팅(파일 기반)
  _layout.jsx
  index.jsx           # 홈(또는 온보딩)
  /auth
    login.jsx
    register.jsx

/components           # 덤프 프레젠테이셔널 컴포넌트
  Button.jsx
  Input.jsx

/lib                   # 공용 유틸·클라이언트
  apiClient.js        # fetch 래퍼(기본 헤더/에러 처리)
  form.js             # RHF+Yup 스키마/헬퍼
  constants.js

/store                 # 전역 상태(Zustand 등)
  useAuthStore.js

/styles
  tokens.js           # 색/간격/라운딩 등 디자인 토큰
  theme.js            # Paper/전역 테마 브리지

/mocks                 # Mock API (MSW)
  handlers.js
  server.js

/assets                # 아이콘/이미지/폰트
```

## Code convention

컨벤션은 기본 Javascript 문법을 따름

## General

- 주석 및 기타 설명은 한국어를 사용할 것
- 가독성이 좋은 코드로 작성할 것
- meaningful variables and function name 사용할 것
- 부가적인 소셜 로그인, 멤버십 결제 등의 페이지 및 기능은 추가하지 말 것
- 각 코드를 페이지별, 기능별로 컴포넌트화 할 것

# 요구사항

- 어플리케이션의 UI는 카카오톡 또는 기본 아이폰 전화번호부 theme 둘 중 하나 선택 가능
- 기본 아이폰 전화번호부의 색은 너무 비비드하지 않은 흰색 바탕에 파란색 아이콘들로 표시
- 단, 현재 테스트 단계의 유저의 경우 id:0000 password:0000

## 랜딩 페이지

- 카카오톡 또는 기본 전화 어플리케이션과 유사한 랜딩 페이지
- 유저는 이메일 또는 핸드폰 번호로 로그인 가능

## 로그인 이후 첫 페이지

- 카카오톡 또는 기본 전화 어플리케이션의 전화번호부와 유사하게 있어야 함
- 유저의 핸드폰 내에 있는 실제 전화를 연동하는게 아니라 유저가 선택하거나 생성한 AI 전화 친구가 등록되어 있는 페이지
- 목록에 있는 친구를 클릭했을 때 전화 걸기 또는 채팅 시작하기 버튼이 있어야 함

## 각 친구 이름을 클릭했을 때

- 목록에 있는 친구는 일반 메신저 어플리케이션처럼 친구의 프로필 사진과 이름, 프로필 상태 메세지가 보임
- 목록에 있는 친구를 클릭했을 때 새로운 창이 stack으롤 쌓이는데, 이 때에 각 친구의 프로필 사진과 이름, 상태 메세지가 보임
- 해당 상태 메세지 하단에 전화 하기 버튼과 채팅 시작하기 버튼이 각각 있음
- stack으로 쌓인 부분을 지우려면 우측 상단에 닫기 버튼(x)를 클릭하면 됨
- 각 친구의 상태 메세지와 이름, 프로필 사진은 설정에서만 고칠 수 있음

## 어플리케이션 하단바

- 어플리케이션 하단바에는 친구 목록, 채팅 목록, 설정 버튼이 있음
- 친구 목록 버튼을 눌렀을 때 '로그인 이후 첫 페이지'와 동일한 페이지로 이동함
- 채팅 목록의 경우 각 친구와 나누었던 대화에 다시 접속 가능하도록 해야함 (기존 메신저들과 동일)
- 설정 버튼을 누르면 새 친구 추가 등의 기능이 존재함
- 채팅 목록에 들어갔을 때 하단에 텍스트 입력 버튼이 있으면 안 됨

## 채팅 목록

- 채팅 목록은 이전 채팅 기록이 목록에 보이면 안 됨
- 열기 버튼 없이 해당 채팅을 클릭했을 때 stack으로 채팅으로 재접속 가능하게 되어야함
- 이전 채팅 내역을 볼 수 있게 해야 함
- 대화를 작성했을 때 내 말풍선은 오른쪽, 상대방의 말풍선은 왼쪽에 위치해야 함
- 상대방 말풍선 가장 왼쪽에는 상대방의 프로필 사진이 보여야 함
